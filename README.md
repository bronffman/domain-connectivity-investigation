# Domain Connectivity Investigation

Компьютер был подключён к внутренней сети компании. Внешний интернет в этом контуре не являлся обязательным, но должны были работать внутренний DNS, домен и корпоративные ресурсы. Windows показывала состояние No Internet, при этом физическое подключение было активно.

В ходе диагностики были проверены:

- DNS-конфигурация;
- доступность контроллера домена;
- маршрутизация;
- ARP-таблица;
- сетевые настройки рабочей станции;
- кабель и порт коммутатора Cisco;
- локальные административные учетные записи.

## Результат

В ходе расследования было установлено, что проблема была связана с некорректной статической сетевой конфигурацией рабочей станции.

После корректировки DNS-серверов и параметров IP-адресации доступ к доменной инфраструктуре был восстановлен.
