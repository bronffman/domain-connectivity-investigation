# Domain Connectivity Investigation

Расследование проблемы доступа рабочей станции к доменной инфраструктуре в изолированном корпоративном сетевом контуре.

В ходе диагностики были проверены:

- DNS-конфигурация;
- обнаружение контроллера домена;
- маршрутизация и ARP;
- параметры IPv4;
- порт и кабель коммутатора Cisco;
- локальные административные учетные записи.

## Root Cause

Причиной проблемы являлась некорректная статическая конфигурация сетевого адаптера:

- отсутствовали DNS-серверы;
- параметры IPv4 не соответствовали используемому сетевому сегменту.

В результате рабочая станция не могла разрешать доменные имена и обнаруживать контроллер домена.

## Resolution

После корректировки IPv4-настроек и настройки DNS-серверов доступ к доменной инфраструктуре был успешно восстановлен.
